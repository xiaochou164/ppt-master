# PPT Master Web 前端重构 PRD（基于现有实现）

## 1. 文档信息

- 文档版本：v1.0
- 基线时间：2026-04-09
- 适用范围：`webapp/static/index.html`、`webapp/static/admin.html`、`webapp/static/app.js`、`webapp/static/app.css` 对应的 Web 控制台前端
- 文档目标：基于“已实现功能”抽象出完整产品需求，作为前端重构输入，不引入未落地能力

---

## 2. 产品定位与目标

### 2.1 产品定位

PPT Master Web 是一个围绕 PPT 生产流水线的“项目工作台”，将项目管理、流程推进、模型配置、生成执行、导出交付、账号管理集中在浏览器端完成。

### 2.2 重构目标

1. 保持现有业务能力不回退。
2. 降低单文件耦合（当前 `app.js` 为超大单文件）。
3. 强化流程状态一致性（前端状态、接口响应、目录产物三者一致）。
4. 提升可维护性（模块边界清晰、可测试、可扩展）。

### 2.3 非目标

1. 不改变底层 PPT 生成脚本能力边界。
2. 不新增“云端协作/多人实时编辑”能力。
3. 不将所有 AI 决策改造成纯前端逻辑。

---

## 3. 用户角色与权限

### 3.1 角色

1. 普通用户（`user`）
2. 管理员（`admin`）

### 3.2 权限矩阵（现状）

1. 普通用户：
   - 可访问主控制台、个人项目、模型配置、模板浏览/上传/删除、账号设置
   - 不可访问 `/admin` 用户管理页
2. 管理员：
   - 拥有普通用户全部能力
   - 可访问用户管理、审计日志、会话强制下线、角色与状态调整

---

## 4. 信息架构与页面结构

### 4.1 页面入口

1. `/login`：登录页（本地登录/注册 或 SSO 引导）
2. `/`：主工作台（7 步流程）
3. `/admin`：管理后台（仅管理员）

### 4.2 主工作台结构

1. 顶部 Hero 区：
   - 用户信息、主题切换、项目管理、模板管理、模型配置、账号设置、退出
2. 项目上下文区：
   - 当前项目关键指标、进度、阻塞信息、快捷动作（刷新/校验）
3. 流程区：
   - 左侧步骤导航（7 步）
   - 中间阶段内容区（按步骤切换）
4. 右侧操作日志区：
   - 运行反馈、错误回显、跳到最新、清空
5. 多个 Modal：
   - 文本模型配置、图片模型配置、项目管理、模板管理、账号设置、SVG 预览、文档预览

---

## 5. 业务流程（端到端）

1. 创建项目（Step 1）
2. 导入来源（Step 2）
3. 模板决策（Step 3）
4. Strategist 分析并保存设计规范（Step 4）
5. 生成图片（可选，Step 5）
6. 生成 SVG 与讲稿（Step 6）
7. 后处理串行执行并导出 PPT（Step 7）
8. 在交付区预览与下载产物

说明：步骤可回看，但有状态门禁；是否“完成/可用/锁定”由项目产物状态驱动。

---

## 6. 功能需求（FR）

## 6.1 认证与会话

1. 系统需支持两种模式：
   - 本地账号模式（登录+注册）
   - Authentik SSO 模式
2. 登录后访问根路径应自动进入控制台；未登录访问受限页面应重定向到 `/login`。
3. 前端需提供退出登录入口，调用后清理会话并跳转登录页。
4. 在 SSO 配置缺失时，登录页需展示缺失项与错误提示。

## 6.2 工作台初始化

1. 首次加载需请求 dashboard 数据，初始化：
   - 用户信息
   - 项目列表
   - 画布格式
   - 后处理步骤
   - 当前文本模型配置
2. 初始化失败时进入“保护态”页面，给出接口错误信息与排障提示。

## 6.3 Step 1 创建项目

1. 支持输入项目名与画布格式创建项目。
2. 支持展示最近项目及筛选（项目名/格式/日期）。
3. 支持切换当前项目。
4. 创建成功后自动刷新状态并进入推荐下一步。

## 6.4 Step 2 导入来源

1. 支持多来源导入：
   - 本地路径（每行一个）
   - URL
   - 粘贴文本（markdown/text）
2. 支持 `move` 导入选项。
3. 支持设置粘贴内容文件名。
4. 支持展示已导入来源并可文档预览。

## 6.5 Step 3 模板选择

1. 支持“自由设计”与“选择模板”两条路径。
2. 支持按分类展示模板及卡片预览。
3. 支持模板应用到项目。
4. 已完成模板决策时显示完成态总结，并给出下一步引导。

## 6.6 Step 4 Strategist 方案确认

1. 支持调用内容分析接口生成建议（八项确认相关字段）。
2. 支持表单编辑并保存为 `design_spec.md`。
3. 表单需覆盖：
   - 画布、页数、受众、风格
   - 主辅强调色
   - 字体与字号
   - 图标方案、图片方案
4. 保存后刷新项目并进入图片步骤。

## 6.7 Step 5 图片生成

1. 需基于设计规范状态决定是否可进入。
2. 支持两种图片工作模式：
   - 依据设计规范识别的图片清单逐张/批量生成
   - 无清单时手动输入 prompt 生成
3. 支持选择宽高比与尺寸。
4. 支持图片模型未配置时的引导。
5. 支持生成结果成功/失败反馈。

## 6.8 Step 6 Executor（页面与讲稿）

1. 支持基于模型配置启动 SVG 生成（SSE 流式进度）。
2. 生成中需展示：
   - 总页数、已完成页数、进度条、最近日志
3. 支持重新生成：
   - 全部重生成
   - 按选定页面重生成
4. 支持删除 SVG：
   - 删除指定页
   - 删除全部（含相关 notes）
5. 在 SVG 生成后，支持生成 `notes/total.md`。
6. 支持展示已生成页面缩略图及预览弹窗。

## 6.9 Step 7 后处理与导出

1. 必须支持且仅支持三步串行执行：
   - `split-notes`
   - `finalize-svg`
   - `export-pptx`
2. 每一步必须展示：
   - 前置条件检查
   - 产出说明
   - 对应命令预览
3. 前置条件不满足时按钮禁用并显示原因。
4. 导出区需展示：
   - 关键文档（design spec / total notes）
   - PPT 文件列表
   - 来源归档
   - 页面预览与文件链接

## 6.10 模型配置（文本）

1. 支持 profile 的新增、编辑、删除、激活切换。
2. 支持后端类型：
   - OpenAI 兼容
   - Gemini
3. 支持：
   - API Key
   - Base URL
   - 模型名（手填或拉取）
4. 支持拉取模型列表。
5. 支持发送短请求测试配置连通性。

## 6.11 模型配置（图片）

1. 支持 profile 的新增、编辑、删除、激活切换。
2. 支持后端类型：
   - Gemini
   - OpenAI (DALL-E)
   - SiliconFlow
3. 支持从全局文本配置复制基础参数。
4. 支持测试图片生成连通性。

## 6.12 项目管理弹窗

1. 支持项目搜索、列表浏览、查看详情。
2. 支持切换当前项目。
3. 支持删除项目（二次确认）。
4. 支持查看项目路径、来源数、页面数、导出文件数。

## 6.13 模板管理弹窗

1. 支持模板搜索与分类筛选。
2. 支持模板详情查看（基础信息、关键词、页面文件、素材）。
3. 支持模板删除（二次确认）。
4. 支持模板上传：
   - 字段：name、label、summary、category、keywords
   - 文件：支持拖拽/选择上传，至少 1 个 SVG
5. 上传/删除后需同步刷新 Step 3 模板缓存。

## 6.14 文档与 SVG 预览

1. 文档预览：
   - 支持 markdown/text 内容弹窗展示
   - 支持新标签页打开
2. SVG 预览：
   - 支持上一页/下一页
   - 支持键盘左右箭头切换
   - 支持新标签页打开
   - 需处理缓存（时间戳避免旧图）

## 6.15 账号设置

1. 支持修改显示名。
2. 本地账号支持修改密码（需当前密码）。
3. SSO 账号需提示密码由外部身份系统维护。

## 6.16 管理后台（Admin）

1. 用户列表：
   - 搜索、角色筛选、状态筛选、来源筛选
   - 分页
2. 用户操作：
   - 修改角色/分组
   - 启用/禁用
   - 强制下线
   - 创建本地账号
   - 本地账号重置密码
3. 用户详情：
   - 基础信息
   - 最近活跃/IP/UA
   - 项目概览
   - 最近审计
4. 审计日志：
   - 动作/资源/时间区间筛选
   - 快捷时间（24h/7d）
   - 分页
   - 复制当前记录
   - 导出 CSV

## 6.17 通用交互

1. 支持主题切换（light/warm/dark）并本地持久化。
2. 页面离开前需检测未保存变更并提示。
3. 所有关键异步操作需有 toast/日志反馈（成功或失败）。
4. Modal 需支持：
   - Backdrop 关闭
   - `Esc` 关闭
   - 焦点陷阱

## 7. 状态与数据需求

### 7.1 前端核心状态（重构建议保留语义）

1. 用户态：当前用户、角色、登录状态
2. 项目态：项目列表、当前项目、项目校验结果
3. 流程态：当前步骤、步骤可用性、推荐下一步
4. 配置态：文本模型配置、图片模型配置
5. 生成态：SVG 生成进度、图片生成状态
6. 管理态：admin 用户列表、审计日志、分页与筛选
7. UI 态：各类 modal 开关、草稿内容、未保存标记

### 7.2 后端接口依赖（按模块）

1. 鉴权/用户：`/api/me`、`/api/auth/status`、`/auth/logout`
2. Dashboard：`/api/dashboard`
3. 项目：`/api/projects`、`/api/projects/{ref}`、`/import`、`/validate`、`/delete`
4. 模板：`/api/templates`、`/api/templates/upload`、`/api/templates/{id}`
5. Strategist/Executor：
   - `/analyze`、`/design-spec`
   - `/generate-svg`、`/regenerate-svg`、`/delete-svg`
   - `/generate-notes`、`/generate-image`
   - `/run-step/{step}`
6. 模型配置：
   - `/api/model-config`、`/api/model-config/models`
   - `/api/image-model-config`、`/api/image-model-config/test`
7. 管理后台：
   - `/api/admin/users`、`/api/admin/users/{id}`
   - `/api/admin/users/{id}/sessions/purge`
   - `/api/admin/audit-logs`、`/api/admin/audit-logs/export`

## 8. 非功能需求（NFR）

1. 安全性：
   - 所有业务接口必须基于会话鉴权
   - 管理接口仅管理员可访问
   - 文件访问需防路径穿越
2. 一致性：
   - 前端步骤状态需由真实项目产物推导，不允许仅靠本地假状态
3. 可用性：
   - 长任务（如 SVG 生成）需可视化进度与错误
   - 错误提示需包含可执行下一步建议
4. 可维护性：
   - 前端需按业务域拆分模块，避免单文件持续膨胀
5. 性能：
   - 项目、模板、日志列表支持筛选与分页
   - 缩略图与预览支持懒加载/缓存控制

---

## 9. 约束与边界

1. 后处理必须串行，不支持前端批处理一键全跑。
2. `export-pptx` 必须依赖 `svg_final` 与拆分讲稿就绪。
3. 图片与文本模型配置按用户隔离，不跨账号共享。
4. 项目与文件按用户隔离访问。

---

## 10. 重构验收清单（建议）

1. 7 步流程与状态门禁行为与现状一致。
2. 所有现有 API 都有对应前端调用路径。
3. 模板管理、项目管理、模型配置三个弹窗功能完整可用。
4. SVG SSE 进度展示、重新生成、删除能力可用。
5. 后处理三步 guard 与执行结果回显可用。
6. 登录/登出/账号设置/管理员后台全链路可用。
7. 预览能力（文档+SVG+键盘导航）可用。
8. 未保存提示、主题持久化、Esc 关闭 modal 可用。

---

## 11. 建议的前端重构模块划分（供实施）

1. `modules/auth`：登录态、用户信息、账号设置
2. `modules/dashboard`：全局初始化、项目上下文、步骤状态机
3. `modules/projects`：项目 CRUD、导入、校验、项目管理弹窗
4. `modules/templates`：Step3 选择 + 模板管理弹窗
5. `modules/strategist`：分析、表单、设计规范保存
6. `modules/images`：图片模型配置、图片生成
7. `modules/executor`：SVG 生成 SSE、重生成、删除、讲稿生成
8. `modules/postprocess`：三步后处理与交付区
9. `modules/admin`：用户管理与审计
10. `modules/ui`：modal、toast、日志面板、主题、焦点管理

该划分应在不改变接口协议与业务规则的前提下推进。
